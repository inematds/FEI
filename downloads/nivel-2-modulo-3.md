# Modulo 3: Fundamentos da Engenharia de Intencao
## Nivel 2 - Programa Completo

**FEI - Formacao em Engenharia de Intencao**

---

## Introducao

Este e o modulo mais importante de toda a formacao. Aqui voce vai aprender a **estrutura completa** para comunicar intencoes com clareza, precisao e profundidade.

A Engenharia de Intencao se baseia em **7 componentes fundamentais** que, quando combinados, transformam prompts vagos em instrucoes estrategicas e eficazes.

### Objetivo deste modulo

Ao final deste modulo, voce tera o **blueprint oficial** da Engenharia de Intencao e sabera construir prompts que funcionam em qualquer contexto.

---

## As 7 Etapas da Engenharia de Intencao

1. **Objetivo** — o que voce quer
2. **Intencao** — para quem, para que, impacto
3. **Criterios de qualidade**
4. **Contexto relevante (e apenas o necessario)**
5. **Estrategia de raciocinio**
6. **Delegacao inteligente (IA deve pedir dados)**
7. **Formato de saida**

---

## Etapa 1: Objetivo — O Que Voce Quer

O objetivo responde: **"O QUE voce quer que a IA produza?"**

### Estrutura do Objetivo

- **Acao:** verbo claro (criar, analisar, resumir, gerar, planejar)
- **Artefato:** o que sera criado (texto, plano, relatorio, estrategia)
- **Escopo:** limites claros (quantos, sobre o que, com que profundidade)

### Exemplo

**Objetivo Vago:**
```
"Me ajuda com marketing"
```

**Objetivo Claro:**
```
"Criar um plano de marketing de 90 dias para lancamento de produto digital"
```

---

## Etapa 2: Intencao — Para Quem, Para Que, Impacto

A intencao responde: **"PARA QUEM e PARA QUE voce esta criando isso?"**

### Componentes da Intencao

- **Audiencia:** quem vai consumir/usar isso
- **Proposito:** qual problema resolve ou objetivo alcanca
- **Impacto desejado:** como voce quer que a pessoa se sinta ou aja

### Exemplo Completo

**Objetivo:** Criar um roteiro de video

**Intencao:**
- **Para quem:** empreendedores iniciantes (sem conhecimento tecnico)
- **Para que:** mostrar como usar IA para automatizar tarefas repetitivas
- **Impacto:** que sintam que e simples e possivel comecar hoje

---

## Etapa 3: Criterios de Qualidade

Criterios definem **"como avaliar se a resposta e boa"**.

### Tipos de Criterios

- **Tom:** profissional, casual, motivacional, tecnico
- **Profundidade:** superficial, moderado, profundo
- **Estilo:** direto, didatico, narrativo, analitico
- **Complexidade:** simples, intermediario, avancado

### Exemplo Pratico

```
Criterios de qualidade:

- Tom: acessivel mas respeitoso
- Evite jargao tecnico desnecessario
- Use exemplos praticos a cada conceito
- Seja direto: maximo 3 frases por ideia
```

### Cuidado!

Nao confunda criterios com formato. **Criterios sao qualitativos** (como deve ser), **formato e estrutural** (como deve estar organizado).

## Etapa 4: Contexto Relevante (E Apenas o Necessario)

Contexto e **tudo que a IA precisa saber** para produzir uma resposta precisa. Mas cuidado: **contexto excessivo confunde**.

### O Que Incluir no Contexto

- **Situacao atual:** onde voce esta agora
- **Dados relevantes:** informacoes especificas necessarias
- **Restricoes:** limitacoes reais (prazo, orcamento, recursos)
- **Historico (se relevante):** o que ja foi tentado

### Exemplo

**Contexto Excessivo:**
```
Tenho 32 anos, trabalho em TI, gosto de ler,
fiz faculdade de Engenharia, moro em SP,
trabalho remoto, tenho 2 gatos...

Preciso de um plano de estudos.
```

**Contexto Relevante:**
```
Contexto:
- Area: desenvolvimento backend
- Disponibilidade: 1h por dia, 5 dias/semana
- Nivel atual: intermediario
- Objetivo: aprender arquitetura de sistemas
```

### Regra do Contexto Minimo

Inclua apenas contexto que **muda a resposta**. Se remover uma informacao e a resposta ideal continua a mesma, essa informacao e desnecessaria.

## Etapa 5: Estrategia de Raciocinio

Em tarefas complexas, voce pode **sugerir uma abordagem** de raciocinio. Mas lembre-se: IA moderna raciocina por padrao.

### Tipos de Raciocinio

**Raciocinio Critico**
- Para analises, avaliacoes, decisoes
- Exemplo: "Analise pros e contras antes de sugerir"

**Raciocinio Criativo**
- Para ideacao, brainstorming, inovacao
- Exemplo: "Explore angulos nao convencionais"

**Raciocinio Tecnico**
- Para solucoes, processos, sistemas
- Exemplo: "Mostre o raciocinio passo a passo"

### Quando usar?

So especifique estrategia de raciocinio em tarefas **realmente complexas** ou quando voce quer ver o processo de pensamento da IA.

## Etapa 6: Delegacao Inteligente

Esta e a etapa mais poderosa: ensinar a IA a **pedir o que ela precisa** antes de responder.

### Por que isso e revolucionario?

Voce nao precisa adivinhar tudo que a IA precisa saber. Ela **pergunta ativamente** o que falta.

### Exemplo de Delegacao

```
Objetivo: Criar plano de marketing

Delegacao:
Antes de criar o plano, me pergunte:
- Qual o orcamento disponivel?
- Qual o publico-alvo especifico?
- Quais canais ja foram testados?
- Qual o prazo para primeiros resultados?
```

### Comparacao

**Sem Delegacao:** Usuario tenta adivinhar tudo que a IA precisa → prompt gigante → informacao excessiva

**Com Delegacao:** IA identifica lacunas → faz perguntas estrategicas → usuario responde so o necessario

## Etapa 7: Formato de Saida

Especifique o formato **apenas quando ele e essencial** para o uso da resposta.

### Quando Especificar Formato

- Integracao tecnica (JSON, CSV, SQL)
- Templates especificos (AIDA, PAS)
- Apresentacoes (slides, relatorios)
- Documentacao (markdown, latex)

### Quando Deixar Livre

- Textos explicativos
- Analises e avaliacoes
- Brainstorming e ideacao
- Conversas exploratorias

### Exemplo de Formato Necessario

```
Formato de saida:
Tabela markdown com 3 colunas:
- Estrategia
- Investimento estimado
- ROI esperado (90 dias)
```

## Blueprint Oficial da Engenharia de Intencao

```
1. OBJETIVO
[Acao] + [Artefato] + [Escopo]

2. INTENCAO
Para quem: [audiencia especifica]
Para que: [problema/objetivo]
Impacto desejado: [como a pessoa deve se sentir/agir]

3. CRITERIOS DE QUALIDADE
- Tom: [profissional/casual/tecnico/motivacional]
- Profundidade: [superficial/moderado/profundo]
- Estilo: [direto/didatico/narrativo/analitico]
- [outros criterios relevantes]

4. CONTEXTO
Situacao atual: [onde voce esta]
Dados relevantes: [informacoes especificas]
Restricoes: [limitacoes reais]

5. RACIOCINIO (opcional)
[Critico/Criativo/Tecnico - apenas se necessario]

6. DELEGACAO
Antes de responder, pergunte:
- [pergunta 1]
- [pergunta 2]
- [pergunta N]

7. FORMATO (opcional)
[Estrutura especifica - apenas se essencial]
```

## 10 Prompts-Modelo Baseados na Estrutura

### Modelo 1: Plano de Acao

```
Objetivo: Criar plano de acao de 30 dias para [objetivo]

Intencao:
Para quem: [eu/equipe/cliente]
Para que: alcancar [resultado especifico]
Impacto: que seja claro e executavel imediatamente

Delegacao:
Antes de criar o plano, me pergunte sobre recursos disponiveis e prioridades.
```

### Modelo 2: Analise Critica

```
Objetivo: Analisar [proposta/ideia/estrategia] de forma critica

Intencao:
Para que: identificar pontos fracos antes de executar
Impacto: decisao informada e consciente dos riscos

Criterios:
- Tom: honesto mas construtivo
- Profundidade: analise detalhada
- Inclua alternativas aos pontos fracos
```

### Modelo 3: Conteudo Educacional

```
Objetivo: Criar explicacao didatica sobre [conceito]

Intencao:
Para quem: [nivel de conhecimento]
Para que: entender e aplicar o conceito
Impacto: sentir que e simples e acessivel

Criterios:
- Use analogias do cotidiano
- Exemplos praticos a cada 2 paragrafos
- Evite jargao sem explicacao
```

### Modelo 4: E-mail Profissional

```
Objetivo: Criar e-mail profissional sobre [assunto]

Intencao:
Para quem: [cargo/relacao]
Para que: [pedir/informar/propor]
Impacto: [resposta esperada]

Contexto:
- Relacionamento: [novo/estabelecido/formal]
- Urgencia: [alta/media/baixa]
```

### Modelo 5: Roteiro de Video

```
Objetivo: Criar roteiro de video de [duracao] sobre [tema]

Intencao:
Para quem: [audiencia]
Para que: [educar/entreter/converter]
Impacto: [acao esperada]

Criterios:
- Gancho nos primeiros 10 segundos
- Linguagem [formal/informal]
- Call to action claro
```

### Modelo 6: Estrategia de Negocio

```
Objetivo: Desenvolver estrategia para [objetivo de negocio]

Intencao:
Para que: [resultado mensuravel]
Impacto: decisao estrategica informada

Delegacao:
Antes de criar a estrategia, me pergunte sobre:
- Recursos disponiveis
- Restricoes conhecidas
- Tentativas anteriores
```

### Modelo 7: Pesquisa Profunda

```
Objetivo: Pesquisar e sintetizar informacoes sobre [tema]

Intencao:
Para que: [aplicacao especifica]
Impacto: compreensao completa e aplicavel

Criterios:
- Profundidade: analise detalhada
- Inclua fontes e referencias
- Destaque insights nao obvios
```

### Modelo 8: Criacao de Curso

```
Objetivo: Estruturar curso de [duracao] sobre [tema]

Intencao:
Para quem: [nivel da audiencia]
Para que: [transformacao esperada]
Impacto: que consigam aplicar imediatamente

Delegacao:
Me pergunte sobre formato, recursos e objetivos de aprendizagem
```

### Modelo 9: Analise de Dados

```
Objetivo: Analisar [tipo de dados] e extrair insights

Intencao:
Para que: [decisao/acao especifica]
Impacto: acao baseada em dados

Criterios:
- Mostre padroes e anomalias
- Conclusoes acionaveis
- Visualizacoes sugeridas
```

### Modelo 10: Melhoria de Processo

```
Objetivo: Otimizar processo de [nome do processo]

Intencao:
Para que: [reducao de tempo/custo/erro]
Impacto: implementacao pratica

Delegacao:
Antes de sugerir melhorias, me pergunte:
- Gargalos conhecidos
- Ferramentas disponiveis
- Restricoes operacionais
```
